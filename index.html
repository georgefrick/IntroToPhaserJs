Elements Of A Game

Game Object

  • Starts up engine
  • Access to common game elements
  • Configure physics
  • Default state
  • Width, height, etc.
The main game object is the entry point to any Phaser game, handling the booting the of engine and providing easy access to both the game objects and the engine.

Game Object


// Basic example, will handle state later.
 var game = new Phaser.Game(800, 600, Phaser.AUTO, 'game');

// Example when game is 'very simply' and has only one state.
 var game = new Phaser.Game(800, 600, Phaser.AUTO, 'game',
                            { preload: preload,
                                create: create,
                                update: update,
                                render: render });

http://docs.phaser.io/Phaser.Game.html

Elements Of A Game

States

  • State Pattern
  • Encapsulates behavior
  • Example states:
    • Boot/Splash
    • Loading
    • Menu
    • Play
    • Pause
    • Game Over
These are all example states, some of which you might handle with a simple flag (Phaser supports pausing natively). You could go with a single, simple state game, or you could create all of these states and more. I will generally create a boot/splash/loading state, a menu state, and a play state. Different games have different appropriate states. For example, many newer games have a stage selection; such as Angry Birds. This stage selection works best as its own state which can startup the game state with the correct state loaded.

Adding States


var Phaser = require('Phaser');
var _ = require('lodash');

var states = {
    boot: require('./states/boot.js'),
    menu: require('./states/menu.js'),
    game: require('./states/game.js')
};

var game = new Phaser.Game(800, 600, Phaser.AUTO, 'game');

_.each(states, function (state, key) {
    game.state.add(key, state(game));
});

// Enter the first state.
game.state.start('boot');

http://docs.phaser.io/Phaser.StateManager.html

I add and advise adding all states up front, but you can add states later.

A Single State


var state = {};

// Called 1st. Use to prep for preload, or skip/change states.
state.init = function () {};

// load assets. Do not use those assets here.
state.preload = function () {};

// Called when preload is complete.
state.create = function () {};

// Called by core game loop.
state.update = function() {};

// Called after rendering of game objects.
state.render = function() {};

http://docs.phaser.io/Phaser.State.html

  • preload: game.load.image('enemyShip', 'images/enemyShip.png');
  • render: game.debug.cameraInfo(game.camera, 500, 32);
  • Will normally load assets in its own state, such as with a splash/loading screen.
  • Splash -> prep for load, and load loading graphics.
  • Load -> Load everything, while displaying load graphics
  • There are other methods you can provide, like shutdown, resize, etc.

Elements Of A Game

Loading Resources

  • Images
  • Sprites
  • Animations
  • Maps / Tiled Data (JSON)
  • Audio
The "resource pipeline" is a varying discussion. Depending on the size and complexity of a game, resources can be a minor bit of work, or it can be one of the major architectural and physical hurdles. For medium sized games, developers face the challenges we face with websites. The user should not be waiting for rendering on a blank screen, nor should they wait an inordinate amount of time for initial feed back when starting the application. For this reason, there are varied approaches to how a game starts up. In our case, we'll show a game that boots up and waits for its resources to load, but doesn't do anything advanced such as a progress bar. By pre-loading the resources, they can all be loaded at once and minimize wait once the game is started.

Boot State


    boot.prototype = {
        preload: function () {
            game.load.image('playerShip', 'images/player.png');
            game.load.image('enemyShip', 'images/enemyShip.png');
            game.load.spritesheet('explosion', 'images/explosion.png', 84, 84);

            game.load.audio('laserSound', 'audio/laser1.ogg');
            game.load.audio('blastSound', 'audio/blast.wav');
        },
        create: function () {
            game.scale.pageAlignHorizontally = true;
            game.scale.pageAlignVertically = true;
            game.scale.refresh();
            this.game.state.start('menu');
        }
    };

http://docs.phaser.io/Phaser.Loader.html

Here we'll load up our resources. In a larger game, we would only load the splash screen resources here. Then we setup the screen, usually according to our device or game design. The setup here is pretty generic for simplicity. You could also wait here until you are ready to show the splash screen. It's turtles all the way down... you could load the company logo and show that logo state until the splash screen resources are ready, then show the splash screen (with or without progress bar) until the resources needed to startup the game are ready. There is also resource batching, you might have a set of resources needed for each stage/level/portion of game and show the loading state while you process the batch. This is what most of us are used to seeing on PC and console games as they load the content.

Waiting On Resources


 update: function () {
    // Wait for music to be ready before starting menu
    if (this.cache.isSoundDecoded('titleMusic') && this.ready === false) {
        this.ready = true;
        this.game.state.start('menu');
    } else {
        // update status bar.
    }
}

http://docs.phaser.io/Phaser.Loader.html

Here we see an approach for waiting on resources. Phaser also has a built in way to do this, by specifying a preloadsprite. We'll see an example of this at the end if we have time.

Sprites

  • Simple Sprites

A Simple Sprite


addPlayer: function() {
    var player = game.add.sprite(400, 550, 'playerShip');
    player.anchor.setTo(0.5, 0.5);
    game.physics.enable(player, Phaser.Physics.ARCADE);
    return player;
}

http://docs.phaser.io/Phaser.Sprite.html

Here we start to see how easy Phaser is. In a few lines of code we've added a sprite to the screen and turned physics on for that sprite. The sprite will use the resource provided as the last string to render (or nothing). You could also create the sprite(s) ahead of time and then add them to the game.

Anchoring

http://docs.phaser.io/Phaser.Sprite.html#anchor

A Group Of Sprites


addLasers: function(imageName) {
    var lasers = game.add.group();
    lasers.enableBody = true;
    lasers.physicsBodyType = Phaser.Physics.ARCADE;
    lasers.createMultiple(5, imageName);
    // One way to change everything in a group is to use 'setAll'
    lasers.setAll('anchor.x', 0.5);
    lasers.setAll('anchor.y', 1);
    lasers.setAll('outOfBoundsKill', true);
    lasers.setAll('checkWorldBounds', true);
    return lasers;
}

http://docs.phaser.io/Phaser.Group.html

Creating sprites as a group can be beneficial because you can move the whole group, change properties of the whole group, etc. In addition to that, you can use a group as a pool, which we're seeing here.

'Facing' A Sprite


player = this.game.add.sprite(200, 200, 'character');


player.animations.add('walk-right', [5, 6, 7, 8]);
player.animations.add('walk-up', [20, 21, 22, 23]);
player.animations.add('walk-down', [35, 36, 37, 38]);


case F_RIGHT: {
    this.player.scale.x = 1;
    if (this.player.body.velocity.x > 0) {
        this.player.animations.play('walk-right', 5, true);
    }
    break;
}
case F_LEFT: {
    this.player.scale.x = -1;
    if (this.player.body.velocity.x < 0) {
        this.player.animations.play('walk-right', 5, true);
    }
    break;
}

A Basic Tween

Looping Sprite Fade


game.add.tween(sprite).to(
                       { alpha: 1 },              // Properties
                       2000,                      // Duration
                       Phaser.Easing.Linear.None, // Ease
                       true,                      // Autostart
                       0,                         // Delay
                       1000,                      // Repeat
                       true);                     // Yo-Yo

http://docs.phaser.io/Phaser.Tween.html#to.

Chaining & Callbacks

Move On Loop


var tween = game.add.tween(sprite).to(
                       { alpha: 1 },
                       2000, Phaser.Easing.Linear.None, true)
                         .to(
                         { x : 1, y: 1 },           // Multiple Properties
                         2000, Phaser.Easing.Linear.None, true);

tween.onStart.add(onStart, this);
tween.onLoop.add(onLoop, this);
tween.onRepeat.add(onRepeat, this);
tween.onComplete.add(onComplete, this);  // Won't fire for looping Tweens

http://docs.phaser.io/Phaser.Tween.html.
